**问题:** 中文显示为 `\uXXXX` 转义字符

---

## 问题描述

发布到微信公众号后，中文翻译全部显示为 `\uXXXX` 格式的转义字符，而不是正常中文。

**错误示例:**
```
\u534a\u4e2a\u4e16\u7eaa\u4ee5\u6765\uff0c\u4e2d\u4e1c\u7684\u77f3\u6cb9\u541b\u4e3b\u56fd...
```

**正确显示:**
```
半个世纪以来，中东的石油君主国...
```

---

## 根本原因

Python 的 `requests.post(json=...)` 参数内部使用 `json.dumps()` 序列化数据，而 `json.dumps()` 默认设置 `ensure_ascii=True`，会将所有非 ASCII 字符（包括中文）转义为 `\uXXXX` 格式。

虽然微信 API 能接收这个格式，但不会自动转换回中文，导致显示为转义字符。

---

## 解决方案

### 修复前（❌ 错误）

```python
def create_draft(access_token, title, content, thumb_media_id):
    url = f"{WECHAT_API_BASE}/draft/add?access_token={access_token}"
    articles = [{
        "title": title,
        "content": content,  # 包含中文
        "thumb_media_id": thumb_media_id,
    }]
    # ❌ 问题：json 参数会自动转义中文
    response = requests.post(url, json={'articles': articles})
```

### 修复后（✅ 正确）

```python
def create_draft(access_token, title, content, thumb_media_id):
    url = f"{WECHAT_API_BASE}/draft/add?access_token={access_token}"
    articles = [{
        "title": title,
        "content": content,  # 包含中文
        "thumb_media_id": thumb_media_id,
    }]
    # ✅ 解决：手动序列化，设置 ensure_ascii=False
    data = json.dumps({'articles': articles}, ensure_ascii=False)
    response = requests.post(url, data=data, headers={'Content-Type': 'application/json; charset=utf-8'})
```

---

## 关键点

1. **使用 `json.dumps(..., ensure_ascii=False)`** - 保留中文字符，不转义
2. **设置正确的 Content-Type** - `application/json; charset=utf-8`
3. **使用 `data` 参数而非 `json` 参数** - 避免 requests 自动序列化
